# ビジネス要件・法務

## 1. 法的・コンプライアンス要件

### 1.1 必須法的文書

#### 利用規約

```yaml
記載必須事項:
  基本情報:
    - サービス名称と概要
    - 提供者情報
    - 利用者の定義
    - 規約への同意方法

  利用条件:
    - 利用資格（年齢制限など）
    - アカウント管理
    - 禁止事項
    - 利用停止・終了条件

  免責事項（最重要）:
    - "本サービスの計算結果は参考値です"
    - "実際の商品寿命は使用状況により異なります"
    - "購入判断は自己責任で行ってください"
    - "計算結果による損害の責任は負いません"

  その他:
    - 知的財産権
    - 準拠法（日本法）
    - 管轄裁判所（東京地方裁判所）
    - 規約の変更
```

#### プライバシーポリシー

```yaml
記載必須事項:
  収集する情報:
    - 匿名の検索データ
    - 使用統計情報
    - Cookieおよび類似技術
    - IPアドレス（ハッシュ化）

  情報の利用目的:
    - サービスの提供・改善
    - 統計データの作成
    - トレンド分析
    - 不正利用の防止

  第三者提供:
    - 原則として提供しない
    - 例外事項の明記
    - 統計データの公開について

  データ管理:
    - 保存期間: 90日
    - 削除方法
    - セキュリティ対策

  問い合わせ先:
    - メールアドレス
    - 対応時間
```

### 1.2 Cookie同意実装

```typescript
interface CookieConsent {
  necessary: boolean;      // 必須Cookie（常にtrue）
  analytics: boolean;      // 分析Cookie
  functional: boolean;     // 機能Cookie
  timestamp: Date;         // 同意日時
  version: string;         // ポリシーバージョン
}

// 実装要件
- 初回訪問時にバナー表示
- 選択的な同意取得
- 同意内容の記録（localStorage）
- 同意の撤回機能
- GDPR準拠の実装
```

### 1.3 GDPR/個人情報保護法対応

```yaml
対応項目:
  データポータビリティ:
    - エクスポート機能（JSON形式）
    - 機械可読形式での提供

  削除要求:
    - 削除申請フォーム
    - 30日以内の対応
    - 削除証明の発行

  アクセス権:
    - 保有データの開示
    - 利用目的の説明
    - 第三者提供の有無

  同意管理:
    - 明示的な同意取得
    - 同意の記録保持
    - 撤回機能の提供
```

## 2. マーケティング戦略

### 2.1 ターゲットユーザー

```yaml
プライマリターゲット:
  年齢: 25-40歳
  特徴:
    - 計画的な消費を心がける
    - 長期的な価値を重視
    - デジタルツール利用に慣れている
    - ミニマリスト志向

セカンダリターゲット:
  若年層（20-25歳）:
    - 初めての大きな買い物
    - 学生・新社会人
    - 予算意識が高い

  中高年層（40-55歳）:
    - 品質重視の買い物
    - 家族のための購入
    - 投資対効果を重視
```

### 2.2 価値提案（Value Proposition）

```yaml
コアバリュー:
  - "買い物を数値化して可視化"
  - "科学的根拠に基づく購入判断"
  - "後悔しない買い物をサポート"

差別化要因:
  - 完全無料・広告なし
  - オフライン対応
  - 科学的根拠の提示
  - プライバシー重視
```

### 2.3 成長戦略

```yaml
Phase 1（0-6ヶ月）:
  目標: 認知拡大
  KPI:
    - MAU: 1,000人
    - 計算実行: 10,000回/月
  施策:
    - SNSでの情報発信
    - ブログ記事作成
    - インフルエンサーコラボ

Phase 2（6-12ヶ月）:
  目標: エンゲージメント向上
  KPI:
    - MAU: 10,000人
    - リピート率: 30%
  施策:
    - 機能追加（履歴、比較）
    - コンテンツマーケティング
    - SEO最適化

Phase 3（12ヶ月以降）:
  目標: マネタイズ検討
  選択肢:
    - プレミアム機能
    - APIの提供
    - B2B展開
```

### 2.4 SEO戦略

```yaml
技術的SEO:

    - 構造化データ（JSON-LD）

コンテンツSEO:

  コンテンツ戦略:
    - 商品別の耐用年数ガイド
    - 購入判断のコツ記事
    - ユーザー事例紹介
```

## 3. リスク管理

### 3.1 想定リスクと対策

```yaml
技術的リスク:
  Supabase障害:
    影響度: 高
    発生確率: 低
    対策:
      - オフラインモード自動切替
      - 静的データでのフォールバック
      - 障害通知の実装

  急激なトラフィック増加:
    影響度: 中
    発生確率: 中
    対策:
      - CDN活用
      - レート制限実装
      - オートスケーリング

ビジネスリスク:
  競合サービスの出現:
    影響度: 中
    発生確率: 高
    対策:
      - 継続的な機能改善
      - ユーザーフィードバック重視
      - 独自性の強化

  法規制の変更:
    影響度: 高
    発生確率: 低
    対策:
      - 定期的な法務確認
      - 利用規約の更新体制
      - 専門家との連携

セキュリティリスク:
  データ漏洩:
    影響度: 高
    発生確率: 低
    対策:
      - 最小限のデータ収集
      - 暗号化の実施
      - 定期的なセキュリティ監査
```

### 3.2 インシデント対応計画

```yaml
対応体制:
  レベル1（軽微）:
    - 担当者対応
    - 24時間以内に解決
    - 記録のみ

  レベル2（中程度）:
    - チーム対応
    - 4時間以内に初動
    - ユーザー通知検討

  レベル3（重大）:
    - 全体対応
    - 即時対応開始
    - ユーザー通知必須
    - 外部公表検討

連絡フロー:
  1. 検知・報告
  2. 影響度評価
  3. 初動対応
  4. 本格対応
  5. 事後対応
  6. 再発防止
```

## 4. KPI設定

### 4.1 主要指標

```yaml
ユーザー指標:
  MAU（月間アクティブユーザー）:
    3ヶ月: 1,000
    6ヶ月: 5,000
    12ヶ月: 20,000

  リテンション率:
    Day 1: 40%
    Day 7: 20%
    Day 30: 10%

エンゲージメント指標:
  計算実行数/ユーザー:
    目標: 月5回以上

  幸福度診断利用率:
    目標: 30%

  平均セッション時間:
    目標: 3分以上

ビジネス指標:
  CAC（顧客獲得コスト）:
    目標: 0円（オーガニック成長）

  LTV（顧客生涯価値）:
    測定方法: 利用頻度×継続期間
```

### 4.2 計測方法

```yaml
ツール:
  Google Analytics 4:
    - ユーザー行動分析
    - コンバージョン追跡
    - ファネル分析

  Supabase Analytics:
    - データベース利用状況
    - APIコール数
    - エラー率

  カスタムダッシュボード:
    - リアルタイムKPI表示
    - 週次/月次レポート
    - 異常検知アラート
```
